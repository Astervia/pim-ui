<objective>
Extend `useDaemonState` to own the app-wide `status` + `peers` reactive spine and ship numeric/time formatting helpers that the Dashboard needs. This plan adds NO user-visible UI. It delivers the pure reactive layer every Wave-2 plan depends on.

Purpose: STAT-04, PEER-01, PEER-05 all require the UI to update reactively from `status.event` / `peers.event` with no polling. Without this plan there is no source of truth for peer lists, no discovered[] buffer for Nearby, and no format helpers for the Identity/Metrics panels. This is the seam that keeps the W1 single-listener contract intact while the downstream UI plans (02-03/04/05) subscribe through one fan-out.

Output:
- `src/hooks/use-daemon-state.ts` — extended with auto-seed (`status`+`peers.discovered`), auto-subscribe (`status.event` + `peers.event`), `discovered[]` buffer, in-place peers merge on `state_changed`/`connected`/`disconnected`, subscription-failure retry-once logic.
- `src/lib/format.ts` — `formatBytes`, `formatDuration`, `formatCount`, `formatShortId` per D-24.
- `src/lib/format.test.ts` — compile-only + runtime assertion tests (TS script style consistent with `rpc-types.test.ts`).
- `src/hooks/use-peers.ts`, `src/hooks/use-status.ts`, `src/hooks/use-discovered.ts` — thin stable-identity selectors so components re-render only when their slice changes.

    - `formatBytes(bytes: number): string`
      - bytes < 1024 → `"{n} B"` (e.g. `formatBytes(512) === "512 B"`)
      - 1024 ≤ bytes < 1024² → `"{n.n} KB"` (1 decimal, e.g. `formatBytes(2048) === "2.0 KB"`)
      - 1024² ≤ bytes < 1024³ → `"{n.n} MB"` (e.g. `formatBytes(4_200_000) === "4.0 MB"` — 4_200_000 / 1048576 ≈ 4.005 → rounds to "4.0 MB"; `formatBytes(4_400_000) === "4.2 MB"`)
      - 1024³ ≤ bytes → `"{n.n} GB"` (e.g. `formatBytes(1_500_000_000) === "1.4 GB"`)
      - Negative / NaN / non-finite → `"0 B"` (defensive; daemon never emits negative, but don't crash)

    - `formatCount(n: number): string`
      - Grouped digits using `Intl.NumberFormat("en-US")` so `formatCount(3847) === "3,847"` and `formatCount(1234567) === "1,234,567"`
      - `formatCount(0) === "0"`

    - `formatDuration(seconds: number): string`
      - seconds < 60 → `"{n}s"` (e.g. `formatDuration(32) === "32s"`)
      - 60 ≤ s < 3600 → `"{m}m {ss}s"` (e.g. `formatDuration(262) === "4m 22s"`, `formatDuration(60) === "1m 0s"`)
      - 3600 ≤ s < 86400 → `"{h}h {mm}m"` (e.g. `formatDuration(15_720) === "4h 22m"`)
      - 86400 ≤ s → `"{d}d {hh}h"` (e.g. `formatDuration(302_400) === "3d 12h"`)
      - negative / NaN → `"0s"`

      - null / undefined / empty string → `"(no id)"` (matches D-20 Nearby row empty-id phrasing)
