## Router + boot sequence

| Option | Description | Selected |
|--------|-------------|----------|
| `AppRoot` wrapper + one-shot `config_exists` invoke | Check once at mount; render FirstRunScreen OR AppShell. Flat, obvious. | ✓ |
| React Router with `/first-run` + `/` routes | Full URL semantics; unnecessary for a one-window Tauri app with no deep-linking. | |
| Conditional rendering inside AppShell itself | Mixes concerns — AppShell should not know about bootstrap state. | |

**Rationale:** AppRoot sits at the right layer (above AppShell, below TunPermissionProvider). Tauri single-window + no URL scheme = one-shot check is honest and simple.

## Platform config path

| Option | Description | Selected |
|--------|-------------|----------|
| User-scope (`~/.config/pim/`, `~/Library/Application Support/pim/`, `%APPDATA%\\pim\\`) | No root required; aligns with standard desktop-app conventions per OS. | ✓ |
| System-scope (`/etc/pim/pim.toml`) | Matches kernel's `DEFAULT_CONFIG` exactly but requires root — unacceptable for Aria. | |
| App-bundle-relative | Works but fights every platform's standards. | |

**Rationale:** PROJECT.md constraint "first-time user can succeed in ≤ 3 interactions" forbids a `sudo` step. User-scope path matches the kernel's `XDG_CONFIG_HOME` fallback on Linux and standard macOS/Windows conventions.

## Default TOML values

| Decision | Selected | Rationale |
|----------|----------|-----------|
| `broadcast = true` | ✓ | Aria expects nearby discovery to work out of the box |
| `bluetooth = false`, `wifi_direct = false` | ✓ | Battery + privacy default; opt-in later in Phase 3 Discovery section |
| `auto_connect = false` | ✓ | Trust must be deliberate — TOFU stays pull-based, never silent |
| `authorization_policy = trust_on_first_use` | ✓ | Matches kernel-study recommendation + PROJECT.md's Briar ethos |
| `require_encryption = true` | ✓ | Default security posture; kernel default anyway |
| `listen_port = 0` | ✓ | OS-assigned port avoids conflicts on dev machines |
| `mesh_ip = "auto"` | ✓ | Daemon picks a CIDR; removes a concept Aria doesn't need |
| `mtu = 1400` | ✓ | Kernel default; no reason to diverge |
| No `[[peers]]` written | ✓ | Peers discovered or added explicitly — empty starts are valid |

## Sidecar crash detection threshold

| Option | Description | Selected |
|--------|-------------|----------|
| ≤500 ms from spawn to Terminated ⇒ crash-on-boot | Long enough to not race with a healthy daemon's startup; short enough to be near-instant from the user's POV. | ✓ |
| ≤2 s threshold | Catches config parse failures AND any slow-path failure. Might misflag slow cold-starts on low-end hardware. | |
| No threshold — always treat Terminated as crash | Breaks Phase 1's normal-shutdown handling. | |

## Config-write atomicity

| Option | Description | Selected |
|--------|-------------|----------|
| Write to `pim.toml.tmp` + fsync + rename | Cheap atomicity guarantee; prevents half-written config if app crashes mid-write. | ✓ |
| Direct write | Simpler but leaves a possible half-written file as a landmine. | |

## Form validation

| Option | Description | Selected |
|--------|-------------|----------|
| Client-side regex `^[A-Za-z0-9_\-\.]+$`, 1-64 chars; inline error on invalid | Matches common hostname-like rules; declarative error. | ✓ |
