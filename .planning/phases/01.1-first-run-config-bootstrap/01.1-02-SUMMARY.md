## Accomplishments

- **`src/lib/rpc.ts`** — added `FirstRunRole` literal union matching the Rust `Role` enum on the wire, `BootstrapConfigArgs` + `BootstrapConfigResult` + `ConfigExistsResult` types, and `bootstrapConfig` + `configExists` typed `invoke()` wrappers. Both wrappers call `@tauri-apps/api/core::invoke` with the exact command names registered by Plan 01.1-01 (`bootstrap_config` / `config_exists`). Zero `listen(` calls added.
- **`src/lib/daemon-state.ts`** — extended `DaemonLastError` to `(RpcError & { kind?: "rpc_error" }) | CrashOnBootError` where `CrashOnBootError` carries the D-19 discriminator. Added `isCrashOnBoot(err: DaemonLastError | null): err is CrashOnBootError` predicate. Added `pickCrashOnBoot(err: DaemonLastError | null): CrashOnBootError | null` — lifts the crash discriminator OUT of `RpcError.data` when the Rust side packed it that way, so UI consumers can branch cleanly on `err.kind === "crash_on_boot"`.
- **`src/lib/rpc-types.test.ts`** — four new compile-only blocks asserting: (a) `bootstrapConfig` accepts exactly `{ nodeName, role }` at the call site; (b) `configExists()` returns `Promise<{ exists, path }>`; (c) `DaemonLastError` narrows on `isCrashOnBoot`; (d) `pickCrashOnBoot` returns `CrashOnBootError | null`. All assertions use `@ts-expect-error` + literal-equality tricks — zero runtime cost, still catches structural drift on every `pnpm typecheck`.
