## Accomplishments

- **`src-tauri/src/daemon/config_path.rs`** — `resolve_config_path()` returns the platform-correct user-scope `pim.toml` path. Linux uses `$XDG_CONFIG_HOME/pim/pim.toml` or `~/.config/pim/pim.toml` fallback via `dirs::config_dir()`. macOS uses `~/Library/Application Support/pim/pim.toml`. Windows uses `%APPDATA%\\pim\\pim.toml`. Never panics; never falls back to root-required paths. Unit-tested per platform branch.
- **`src-tauri/src/daemon/default_config.rs`** — `render_default_config(node_name: &str, role: Role) -> String` produces the D-16 TOML template verbatim. `Role::JoinTheMesh` → `roles.gateway = false`; `Role::ShareMyInternet` → `roles.gateway = true`. All other fields are D-16 defaults (broadcast on, BLE/WFD off, auto_connect off, TOFU auth, require_encryption on, listen_port 0, mtu 1400, mesh_ip auto). Unit-tested with both role values + node-name interpolation.
- **`src-tauri/src/rpc/commands.rs`** — Two new `#[tauri::command]` handlers:
  - `bootstrap_config(node_name: String, role: Role) -> Result<BootstrapConfigResult, String>` — resolves path, creates parent dir at 0o700 on Unix, renders TOML via `render_default_config`, writes atomically (tmp + fsync + rename per D-14), returns `{ path: String }`. Structured error on each failure mode.
  - `config_exists() -> Result<ConfigExistsResult, String>` — stat-checks the resolved path, returns `{ exists: bool, path: String }`. fs errors (D-22) return `exists: false` gracefully.
- **`src-tauri/src/daemon/sidecar.rs`** — `spawn()` signature extended with `on_crash_on_boot: F where F: Fn(CrashOnBootInfo) + Send + Sync + Clone + 'static`. Inside the inner tokio task, `spawned_at: Instant::now()` captured before the event loop; each stderr chunk appended to a 2 KiB ring buffer; when `Terminated` arrives, if `spawned_at.elapsed().as_millis() < 500`, the closure is invoked with `CrashOnBootInfo { exit_code, signal, stderr_tail, elapsed_ms, config_path }`. ≥500 ms keeps the existing `tracing::warn!` path unchanged. Outer `spawn()` still returns `Ok(())` the moment the child process is alive (Phase 2 Plan 02-01 contract preserved).
- **`src-tauri/src/daemon/state.rs`** — New `CrashOnBootInfo` struct + `DaemonConnection::report_crash_on_boot(self: Arc<Self>, app: AppHandle, info: CrashOnBootInfo)` method. Builds an `RpcError` with code `-32000`, human message `"pim-daemon exited in {elapsed_ms} ms during startup"`, and `data: Some(serde_json::json!({ kind: "crash_on_boot", path, stderr_tail, elapsed_ms, exit_code, signal }))` — the D-19 discriminator. Cancels the handshake watchdog if armed. Delegates to the existing `set_state(Error, Some(err), None, None)` which already emits `daemon://state-changed`. `set_state` visibility widened to `pub(crate)` for cross-module reach; no external API change. Locked unit test (`crash_on_boot_data_shape_is_d19_compliant`) asserts the exact JSON shape the TS side of Plan 02 will discriminate on.
- **`src-tauri/src/daemon/mod.rs`** — added `pub mod config_path;` and `pub mod default_config;`.
- **`src-tauri/src/lib.rs`** — registered `rpc::commands::bootstrap_config` and `rpc::commands::config_exists` in `tauri::generate_handler!`. Existing 6 Phase-1 commands untouched.
- **`src-tauri/Cargo.toml`** — added `dirs = "5"` for XDG-compliant path resolution on Linux. No other dep changes.
