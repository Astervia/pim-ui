  <action>
    Each section component has the same shape:
    1. Read `base` from `useSettingsConfig()`; extract defaults via `getPath(base, wireKey)`
    2. Init `react-hook-form` with those defaults; track `formState.isDirty`
    3. Call `useSectionSave(sectionId, form)` — note the `form` param is REQUIRED per checker Blocker 1 (the hook writes form.formState.isDirty into the use-dirty-sections module atom via useEffect)
    4. Render `<CollapsibleCliPanel>` with a summary computed from the current form values
    5. If `useSectionRawWins(sectionId).rawWins === true`, render `<RawWinsBanner />` at the top of the body
    6. If `sectionBannerError !== null` from useSectionSave, render it above the fields
    7. Render the fields (per section spec below) — each field wrapped in FormField/FormItem with label + wire-name tooltip + control + FormMessage (reads from `fieldErrors[wireKey]`)
    8. `<SectionSaveFooter>` passes `dirty={formState.isDirty}`, `state={saveState}`, `onSave={() => save(formValuesKeyedByWirePath)}`

    Form values are keyed by the fully-qualified wire path (`"transport.listen_port"`, `"discovery.broadcast"`, etc.) so `assembleToml` can directly `setPath()` them.

    Simplest pattern: register each field with a SHORT local name (e.g. `name`, `listen_port`), then at submit time construct the wire-keyed values object:
    ```typescript
    const onSave = () => {
      const values = form.getValues();
      save({
        "transport.interface": values.interface,
        "transport.mtu": Number(values.mtu),
        "transport.mesh_ip.mode": values.mesh_ip_mode,
        "transport.mesh_ip.value": values.mesh_ip_value,
        "transport.listen_port": Number(values.listen_port),
      });
    };
    ```

    ### TRANSPORT — `transport-section.tsx` (CONF-03)

    Summary: `interface {iface} · mtu {mtu} · mesh_ip {mode} · port {listen_port}`
    Fields (verbatim labels per 03-UI-SPEC §Form field labels):
    - `Interface name` → wire `transport.interface`, text Input
    - `MTU` → wire `transport.mtu`, number Input (min=576 max=9216 step=1)
    - `Mesh address mode` → wire `transport.mesh_ip.mode`, RadioGroup with two options:
      - `static` → label `Static — I set the address`
      - `auto` → label `Automatic — pim picks an address`
    - `Your address on the mesh` → wire `transport.mesh_ip.value`, text Input — **render only when `mesh_ip.mode === "static"`** (conditional on a `watch()` call)
    - `Listen port` → wire `transport.listen_port`, number Input (min=1 max=65535)

    Pending-restart token: if `usePendingRestart("transport").fields.length > 0`, pass a `pendingRestartToken={<span className="font-mono text-accent">· ⚠ pending restart: {fields.join(", ")}</span>}` prop to `<CollapsibleCliPanel>`.

    ### DISCOVERY — `discovery-section.tsx` (CONF-04)

    Summary: `broadcast {on/off} · BLE {on/off} · wifi_direct {on/off} · {N} trusted peer{s}`
    Fields (all Switch; labels VERBATIM per 03-UI-SPEC):
    - `Broadcast discovery` → wire `discovery.broadcast`
    - `Bluetooth discovery` → wire `discovery.bluetooth`
    - `Wi-Fi Direct discovery` → wire `discovery.wifi_direct`
    - `Auto-connect to discovered peers` → wire `discovery.auto_connect`

    The `{N} trusted peer{s}` portion of the summary pulls from the parsed `security.allow_list[]` if present in base; use `(base?.security as any)?.allow_list?.length ?? 0`. A permissive cast is acceptable — the schema diff logic catches drift.
